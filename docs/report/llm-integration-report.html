三、SDK 接入评估

SDK 可用性总结

提供商官方 Node.js SDKOpenAI SDK 兼容当前接入方式评估
Qwen无是fetchOpenAI SDK 为官方推荐方式，当前 fetch 实现等效
Moonshot无是fetch官方明确推荐 OpenAI SDK，当前 fetch 实现等效
Zhipu无（仅 Python zai-sdk）是fetchNode.js 无官方 SDK，OpenAI SDK 为推荐替代
Doubao@volcengine/ark-runtime是官方 SDK已接入官方 SDK，获得连接池、重试、超时控制
MiniMax无是（OpenAI + Anthropic）fetch官方推荐 OpenAI SDK，当前 fetch 实现等效
Hunyuan无（腾讯云 SDK 为签名方式）是fetchTokenHub 平台官方推荐 OpenAI SDK，当前 fetch 实现等效
结论：12 家 LLM 提供商均完全兼容 OpenAI API 格式。豆包已接入官方 Node.js SDK @volcengine/ark-runtime，其余 5 家国产 LLM 提供商官方均推荐使用 OpenAI SDK 调用，当前项目采用 fetch 直接调用，与 OpenAI SDK 功能等效，且具有以下优势：
  • 无额外依赖，减少包体积
  • 统一代码模式，11 家提供商实现结构一致
  • 完全控制请求体结构，便于添加提供商特有参数（如 enable_thinking、reasoning_split）
  • 避免 SDK 版本升级带来的 breaking change 风险

四、本次修复内容

  • Kimi 模型版本统一 — kimi-k2.6 → kimi-k2.7-code（2026-06-12 发布，4 处：客户端构造函数、工厂 switch、getDefaultModel、连通性脚本）
  • 豆包模型 ID 格式修复 — doubao-seed-2.0-pro → doubao-seed-2-0-pro-260215（连字符+日期后缀，符合火山引擎官方格式）
  • 豆包接入官方 SDK — 从裸 fetch 迁移至 @volcengine/ark-runtime 官方 Node.js SDK，获得连接池、自动重试、超时控制等生产级特性
  • 混元平台迁移 — hunyuan-turbos-latest（2026-06-26 下线）→ hy3-preview（TokenHub 新平台，MoE 架构），Base URL 改为 tokenhub.tencentmaas.com/v1
  • MiniMax Base URL 修复 — 使用国内版 api.minimaxi.com/v1（非国际版 api.minimax.io）
  • MiniMax 推理参数 — 添加 reasoning_split: true 分离思考内容到 reasoning_details 字段
  • Qwen 推理参数 — 添加 enable_thinking: true 启用 Qwen3.x 深度思考模式
  • GLM 推理参数 — 添加 enable_thinking: true + reasoning_effort: 'max' 启用 GLM-5.2 思考模式
  • Kimi thinking 参数 — 添加 thinking: { type: 'enabled' } 显式启用 K2.7-Code thinking 模式
